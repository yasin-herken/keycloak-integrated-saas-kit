# Autonomous Agents Overview

This document outlines the architecture and responsibilities of the autonomous AI agents utilized within the OpenSpec framework for managing and executing the `enterprise-saas-kit` project.

These agents are designed to handle specific domains of the software development lifecycle, ensuring that the specification remains the single source of truth while automating code modifications and reviews.

## 1. Specification Agent (`spec-agent`)

The core intelligence responsible for maintaining the structural integrity and logic of the project's specification files.

*   **Role:** Architect & Requirements Analyst
*   **Responsibilities:**
    *   Parses incoming user requirements or feature requests.
    *   Translates requirements into formal Spec-Driven Development formats.
    *   Ensures that new requirements do not conflict with existing architectural constraints (e.g., maintaining the strict separation of Keycloak for Identity Management).
    *   Generates and updates the core `.spec` files (e.g., `api.spec.yaml`, `auth.spec.yaml`).
*   **Triggers:** Activated when a new high-level requirement is introduced or when a structural change is needed in the business logic.

## 2. Code Generation Agent (`codegen-agent`)

The execution engine that translates formal specifications into concrete, production-ready code.

*   **Role:** Full-Stack Developer
*   **Responsibilities:**
    *   Reads the updated `.spec` files generated by the `spec-agent`.
    *   Generates Spring Boot controllers, services, and repository layers based on API specifications.
    *   Implements secure Keycloak token validation (Bearer Token approach) without writing custom authentication logic.
    *   Scaffolds the "dumb" frontend components based on the 5-dollar UI template structure, ensuring they only map to the defined backend endpoints.
    *   Generates Flyway/Liquibase migration scripts for database schema changes.
*   **Triggers:** Activated whenever a `.spec` file is updated and committed.

## 3. Infrastructure & Deployment Agent (`infra-agent`)

The operations specialist ensuring the application is deployed correctly and securely.

*   **Role:** DevOps Engineer
*   **Responsibilities:**
    *   Maintains and updates `docker-compose.yml` for local development (coordinating PostgreSQL, Keycloak, and the Spring Boot API).
    *   Generates and validates Kubernetes manifests (Deployments, Services, Ingress).
    *   Ensures ArgoCD application definitions are in sync with the desired state of the repository.
*   **Triggers:** Activated when architectural changes require new infrastructure components (e.g., adding a Redis cache) or when deployment configurations are modified.

## 4. Security & Audit Agent (`security-agent`)

The gatekeeper that ensures the generated code and infrastructure adhere to the defined security standards.

*   **Role:** Security Auditor
*   **Responsibilities:**
    *   Scans generated Spring Boot code to ensure XSS protection (Jackson configurations) and CSRF mitigation (Bearer token validation) are correctly implemented.
    *   Validates CORS configurations to prevent wildcard origins.
    *   Checks Bucket4j rate limiting implementations.
    *   Verifies that no plain-text passwords or secrets are hardcoded in the infrastructure files.
*   **Triggers:** Runs automatically as a pre-commit hook or during the CI pipeline on all generated code before it is merged.

## Execution Workflow

1.  **Requirement Input:** User provides a natural language requirement to the `spec-agent`.
2.  **Specification Update:** `spec-agent` translates this into a formal `.spec` update.
3.  **Code Synthesis:** `codegen-agent` and `infra-agent` read the new spec and generate the corresponding application code and infrastructure files.
4.  **Security Review:** `security-agent` reviews the output against established enterprise constraints.
5.  **Finalization:** Approved code is committed and ready for manual review or automated deployment.
